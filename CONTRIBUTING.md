# Contributing

Catalyst Finance welcomes improvements that make financial reasoning more transparent, reproducible, and responsible.

## Priorities

Good contributions should improve one or more of the following:

- clearer assumptions
- better scenario documentation
- source and method traceability
- reproducible calculations
- reviewable exports
- responsible boundaries around financial interpretation

## Local checks

```bash
python3 -m pytest -q
```

## Style

Use clear variable names, explicit units, and plain-language interpretation notes. Avoid hiding assumptions inside polished outputs.

## Boundaries

Do not add features that imply automatic investment advice, financial recommendation, tax advice, legal advice, assurance, or guaranteed outcomes.
