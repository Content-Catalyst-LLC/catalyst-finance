# Catalyst Finance v1.0.1 — Repository Integrity and CI Repair

Release date: 2026-07-17

## Purpose

Convert the initial prototype into a clean, installable, testable repository before expanding the financial model.

## Delivered

- Established `1.0.1` across the Python package, API, WordPress plugin, manifest, schema, examples, and documentation.
- Replaced the root elasticity CLI with a real ASGI application entry point.
- Moved demand and elasticity functions to `catalyst_finance.elasticity` and added the `catalyst-finance-elasticity` command.
- Added `pyproject.toml`, Python 3.10–3.13 support, dependencies, optional plotting support, and console entry points.
- Added a FastAPI application factory and working `/healthz` and `/api/v1/version` routes.
- Preserved the previous scenario command through `python/catalyst_finance_core.py`.
- Replaced duplicate GitHub Actions definitions with one release-oriented matrix.
- Added static checks, tests, JSON validation, PHP syntax, JavaScript syntax, deterministic plugin packaging, reproducible examples, and smoke tests.
- Moved generated elasticity CSV files and plots into explicit example directories.
- Added a release contract that detects stale versions, missing paths, duplicate workflows, transient caches, unexpected archives, invalid examples, and broken packages.
- Repaired Ruff 0.15 compatibility by removing redundant explicit `None` defaults from elasticity result lookups.

## Compatibility

The scenario input and result calculations remain compatible with the initial prototype. The legacy elasticity modes remain available under a named command rather than root `app.py`.
