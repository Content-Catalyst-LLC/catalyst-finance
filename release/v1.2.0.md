# Catalyst Finance v1.2.0

## Persistent Scenarios and Workspace Management

Released: 2026-07-17

### Purpose

Move Catalyst Finance from a disposable screening calculator to a reusable, versioned scenario workspace without changing the underlying annual screening mathematics.

### Delivered

- Canonical `FinanceWorkspace`, `WorkspaceProject`, `WorkspaceScenario`, `ScenarioRevision`, `WorkspaceExport`, and `ScenarioTemplate` contracts.
- Immutable identifiers and ordered append-only revision chains.
- Workspace defaults for currency, locale, timing, basis, and model references.
- Atomic JSON and SQLite repositories implementing one protocol.
- Explicit save, autosave recovery, recovery commit, and interrupted-work protection.
- Create, duplicate, rename, archive, restore, import, export, and delete lifecycle operations.
- Complete identifier-preserving workspace export/import.
- Five built-in scenario templates.
- Workspace CLI and API routes.
- Multi-scenario browser workspace with local persistence and public read-only mode.
- v1.1.0-to-v1.2.0 migration in Python and JavaScript.
- Versioned workspace schemas and a reproducible two-revision example export.

### Acceptance gate

A user can create a workspace, create multiple scenarios, close and reopen it through either repository adapter, add a revision without overwriting history, recover an interrupted autosave, and export/import the complete workspace without changing any workspace, project, scenario, or revision identifier.

### Boundary

The release continues to provide educational decision-support software only. Workspace persistence and revision history do not convert screening outputs into financial, investment, tax, accounting, legal, fiduciary, lending, procurement, funding, or assurance advice.
