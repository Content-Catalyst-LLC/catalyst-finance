# Catalyst Finance v1.4.0 — Scenario Comparison, Sensitivity, and Break-Even

## Purpose

Show how financial conclusions change across alternatives and assumptions without hiding the underlying scenario revisions.

## Added

- A versioned `catalyst-finance.comparison` contract requiring at least three alternatives.
- Base, downside, upside, and custom alternative classifications.
- Source workspace, scenario, revision, and revision-number traceability.
- Aligned metrics, baseline deltas, per-metric ranks, weighted rankings, and financial Pareto dominance checks.
- One-way and two-way deterministic sensitivity analysis.
- Bounded break-even and threshold analysis for top-level assumptions, line-item values, category values, multipliers, and implementation delays.
- Tornado chart data and crossover-point detection.
- One-assumption-at-a-time NPV driver explanations with interaction warnings.
- Versioned comparison definitions inside persistent workspaces.
- JSON, CSV, Markdown, and printable HTML comparison exports.
- FastAPI, CLI, Python, browser, and WordPress comparison surfaces.
- Migration of v1.3.0 screening and cash-flow inputs into the v1.4.0 contracts.

## Review boundary

Rankings and dominance checks use selected financial model outputs only. Non-financial caveats remain visible and are never converted into an undisclosed score. Sensitivity and break-even outputs describe the declared model, not certainty about real-world outcomes.

## Validation gate

The release must compare three or more revision-linked alternatives; reproduce sensitivity and threshold runs; preserve comparison revision history; validate generated schemas and exports; match Python and browser comparison-core outputs; and pass Python, JavaScript, PHP, packaging, and isolated installer checks.
