# Catalyst Finance Methodology

Catalyst Finance treats financial analysis as a structured reasoning workflow rather than a black-box recommendation engine.

## Core principle

Every financial conclusion should remain connected to the assumptions and inputs behind it.

```text
initiative -> costs -> benefits -> assumptions -> risk adjustment -> interpretation -> review note
```

## Scenario elements

A finance scenario should state:

- initiative name
- capital cost
- grant, rebate, or external funding
- annual savings or avoided cost
- annual operating cost
- time horizon
- discount rate
- carbon-price assumption, if used
- annual emissions reduction, if used
- evidence confidence
- implementation risk
- review status

## Interpretation rules

The engine reports NPV, payback, ROI-style estimates, benefit-cost ratio, carbon cost per ton, and risk-adjusted decision notes. These are screening outputs, not final recommendations.

## Responsible use

Use outputs to structure review. Do not use them as investment advice, legal advice, tax advice, procurement advice, lending advice, or assurance.
